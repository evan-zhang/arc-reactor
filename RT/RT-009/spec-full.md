# RT-009 Spec Full: 冲突对撞机 (Conflict Collider)

## 📌 1. 业务目标
解决个人知识库随时间推移必然产生的**“认知冲突”**问题。当新知识进入时，不再是盲目地追加，而是主动与已有档案进行逻辑对撞。如果发现新旧观点打架（例如对某项技术的评价从“推荐”变成了“弃坑”），系统必须主动标记并提请主人最终裁决。

## 🎯 2. 技术规格

### 2.1 冲突探测脚本 `scripts/conflict-collider.py`
一个作为 `Ingest` 预处理器或后置校验器的工具。
- **对撞逻辑**：
    1.  脚本接收新要摄入的文档流。
    2.  提取其中的 `[[wiki-links]]`，检测同名实体文件是否已存在于 `wiki/entities/`。
    3.  若存在，将**新文档内容**与**既有实体卡片内容**进行拼接。
    4.  调用顶级推理模型（如 gpt-4o），执行专门的“批判性审查”指令集。
- **审查维度**：
    - **事实冲突**：数据、版本、日期的前后矛盾。
    - **逻辑冲突**：推导结论的截然相反。
    - **价值变动**：个人评价偏好的重大转向。

### 2.2 交互形式
- **静默匹配**：如果没有冲突，静默通过，不做拦截。
- **碰撞警报**：一旦发现冲突，输出一个详细的 `[CONFLICT_ALERT]` 包裹块，包含：
    - `Old Vision`: 旧文档的关键论点。
    - `New Vision`: 新文档的对立点。
    - `Confidence`: 算法判断冲突的置信度。

- 支持 `--force-override` 参数，允许用户在知晓冲突的情况下强制覆盖旧观点。
