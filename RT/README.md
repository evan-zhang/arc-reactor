# OpenClaw RT (Research Ticket) 治理规范

本目录是所有研发任务（RT）的指挥与档案中心。

## 📂 核心存储原则：单项目内闭环

ARC Reactor 是一个独立的、功能完备的 Skill 项目。我们遵循以下归档与管理原则：

1.  **任务即目录**：每一个研发任务（RT）必须落位在独立的文件夹中，例如 `RT/RT-NNN/`。
2.  **档案永久留存**：RT 任务档案是项目演进的历史记录。任务完成后（Status: done），档案应**永久保留**在本目录下，不再物理迁移至其他仓库。
3.  **标准化命名**：
    *   主文档：根据任务 Profile 命名为 `spec-lite.md` 或 `spec-full.md`。
    *   支持附件：所有的计划库、交付文档、验收细则均存放在对应文件夹下。
4.  **主控索引**：`RT/index.yaml` 是所有研发任务的唯一权威索引，记录了任务的状态、负责人及关键元数据。

## 📊 任务状态流转

- **todo**: 待启动。
- **doing**: 进行中，包含已批准的方案。
- **done**: 已完成，且通过了验收。
